Thought of Galaxy

プロンプトを、思考の銀河として扱う。

AI共創 Local Nova は、Ollama のローカルモデルと対話するための自作チャット環境です。 プロンプト、知識、記憶、ワークフローを文脈に応じて切り替えながら、手元のAIと共創するためのコクピットとして設計されています。

瞬時に入れ替える

定型プロンプト、ワークフロー、プロジェクト知識をすばやく呼び出し、会話の目的に合わせてAIへ渡す文脈を組み替えます。

  • 手元のモデルとプロンプトを育てる。
  • 会話の目的に合わせて文脈の順序を選ぶ。
  • 知識、記憶、ワークフローを継続的に扱う。
  • 保存した思考の断片を、必要な瞬間に呼び戻す。

思考の銀河を、そのまま操作画面へ。

Local Nova のUIは、チャット、プロジェクト、記憶、プロンプト、ワークフローを一つの操縦席に集約します。 左で会話とプロジェクトを選び、中央で対話し、下部のボタンから思考支援機能をすばやく呼び出せます。

Composer

下部コントロール

添付、検索、定型プロンプト、メモリー、エージェント、ワークフロー、プレ出力をすぐ起動できます。

Base Controls

基本操作と会話の土台

モデル、セッション、サイドバーを切り替えながら、会話の入口を整えます。

Sidebar Controls

モデル、プロジェクト、セッションをすばやく切り替える。

左サイドバー上部には、使用モデル、設定、Project、Session Prompt を集約。対話の土台を切り替える操作を常に近くに置いています。

Model Selector

手元のOllamaモデルを、その場で選ぶ。

インストール済みモデルを一覧から選び、会話の目的に合うモデルへ切り替えます。ローカルAIを使い分けるための入口です。

Session Prompt

会話ごとに、AIへ渡す前提を整える。

Session Prompt は現在の会話に適用する指示を保存する場所です。文章のトーン、役割、出力方針を会話単位で切り替え、プロジェクト知識やメモリーと合わせて文脈を設計できます。

Custom Prompt

カスタムプロンプト

セッションプロンプトとカスタムプロンプトを使い分け、指示を効かせる範囲を調整できます。

Session Prompt

そのスレッド内のみに作用するプロンプト。

セッションプロンプトは、現在開いているスレッド内だけに作用する指示です。会話ごとの前提、口調、役割、出力方針を一時的に切り替えたい時に使います。

Custom Prompt

スレッドを跨ぎ、全体に作用するプロンプト。

カスタムプロンプトは、スレッドを跨いで全体に作用する指示です。長期の口調、優先順位、守ってほしい姿勢、継続中の前提などを安定して渡せます。

System Prompt

システムプロンプト

設定画面から Ollama の ModelFile を作成し、ターミナルでコマンドを打たずにオリジナルモデルを作れます。

System Prompt

Ollama の ModelFile を、設定画面で作る。

設定 ＞ システムプロンプトでは、派生モデルへ反映する固定SYSTEMを役割ごとに整理できます。入力した内容は ModelFile の SYSTEM としてまとめられ、ターミナルでコマンドを打たなくてもモデル作成の準備ができます。

Derived Model

設定 ＞ システムプロンプト ＞ 派生モデルで、オリジナルモデルを作る。

システムプロンプトから作成した派生モデルは、派生モデル画面で確認できます。ベースモデルに固定SYSTEMを反映した、用途別のオリジナルモデルとして保存し、必要に応じて切り替えられます。

Model Selector

作成したモデルを、通常のモデル一覧から選ぶ。

作成したオリジナルモデルは、通常の Ollama モデルと同じようにモデル選択欄へ表示されます。用途に合わせたSYSTEMを焼き込んだモデルを、会話開始時にそのまま選択できます。

High-Speed Model

思考を抑えた、高速応答用モデルとしてチューニングする。

Gemma4 は通常 Thinking を出しやすいモデルですが、固定SYSTEMを反映した派生モデルとして調整することで、スピード重視の応答に寄せられます。この例では High-Speed-Model として、思考を長く出さず、すぐ必要な回答へ進むモデル設計にしています。

Temperature

温度設定

設定 ＞ 温度設定から、回答のランダム性や創造性を制御する Temperature パラメータを調整できます。

Temperature Control

生成温度を、設定画面から調整する。

Temperature パラメータは、通常 0.0 から 1.0、モデルによってはそれ以上の範囲で調整される数値です。設定 ＞ 温度設定で ON にすると、指定した温度をすべての生成へ送れます。

Creative Output

高温では、既存の話から離れた創造的な回答を出しやすい。

たとえば温度設定が 2 のように高い場合、既存の昔話とは違う展開を作りやすくなります。創造的な回答を期待する場合は高温に、既存の知識を安定して引き出す場合は低温に設定します。

Context Window

コンテキスト送信量

会話履歴として送るコンテキスト量を、モデルや環境に合わせて調整できます。

Context Window

モデルに合わせて、送信する文脈量を決める。

コンテキスト送信量は、会話履歴としてモデルへ送る文脈の量です。小規模言語モデルでは送信量が大きすぎると生成が重くなったり、動きが悪くなる場合があります。まずはプリセットから選び、使うモデルの性能と作業内容に合わせて送信量を調整します。

Prompt Library

頻度の高いプロンプトを保存して呼び出す

設定画面で保存した定型プロンプトを、チャット欄の P ボタンからすぐ差し込めます。

Prompt Library

頻度の高いプロンプトを、設定画面で保存する。

よく使う役割指定、出力形式、思考ルールを定型プロンプトとして保存。名前と本文を登録しておくことで、毎回書き直す手間を減らします。

Quick Insert

保存したプロンプトを、チャット欄からすぐ呼び出す。

チャット下部の P ボタンから保存済みプロンプトを開き、クリックで入力欄へ差し込みます。送信前に編集できるため、その場の文脈に合わせて微調整できます。

Search System

検索で、小規模モデルの知識不足を補う

検索先と検索クエリ生成プロンプトを選び、必要な外部情報をAIへ渡してハルシネーションを抑えます。

検索なしでは、知らない情報を推測しやすい。 小規模言語モデルに入っていない固有名詞や最新情報は、誤った回答につながる場合があります。
検索ボタンで、外部情報を会話へ接続する。 必要な時だけ検索を走らせ、検索プロセスを開いて確認できます。
根拠を取り込み、回答を現実側に寄せる。 取得した検索結果を補助文脈として渡し、参照元を確認しながら回答します。
ハルシネーション
Hallucination Risk

知らないことを、知っているように答えさせない。

小規模言語モデルは、内部知識にない人物、固有名詞、専門情報で推測が混ざりやすくなります。検索を使うことで、回答前に外部情報を補助文脈として渡せます。

Search Toggle

チャット欄から、検索をすぐ有効化する。

入力欄の検索ボタンをONにすると、質問に対して外部検索を実行します。会話の流れを止めず、必要な時だけ文脈を拡張できます。

Search Status

検索中の状態を、回答欄で確認できる。

検索が走っている時はステータスとして表示されます。必要に応じて開閉し、検索の進行だけを軽く確認できます。

Search Process

検索プロセスを開き、何を探しているか確認する。

検索先、検索方式、検索語、取得件数、透明性ログをまとめて確認できます。AIがどの情報を探しているかを隠さず見せる設計です。

Search Providers

検索先を、用途に合わせて選択する。

一般知識、論文、医学・生命科学、自前検索など、必要な検索先を選べます。情報の種類に合わせて参照元を切り替えられます。

Query Control

検索クエリの作り方も、プロンプトで制御する。

キーワード検索、クエリ検索、ディープ検索を選び、関連クエリを生成するプロンプトも調整できます。検索の精度をアプリ側で設計できます。

Grounded Answer

検索結果を取り込み、参照元つきで回答する。

検索で得た情報を補助文脈として使い、回答の根拠を確認しやすくします。小規模モデルでも、知らない領域を検索で補強できます。

Memory

過去履歴を、必要な量だけ文脈へ戻す

M ボタンで他スレッド履歴を読み込み、設定画面で読み込むスレッド数を調整できます。

Memory Toggle

Mボタンで、過去履歴を補助文脈として読み込む。

チャット欄の M ボタンを ON にすると、他スレッドの履歴を現在の会話へ補助文脈として渡せます。過去のやり取りを使い、継続した共創をしやすくします。

Memory Settings

読み込むスレッド数を、目的に合わせて調整する。

設定画面では、M ON 時に読み込む他スレッド数をコントロールできます。文脈を広げたい時は増やし、集中したい時は絞ることで、コンテキスト消費を管理できます。

Agent Assist

エージェント補助機能

自動実行中の見守りと完了確認を支える、エージェントモード向けの補助機能です。

Agent Alarm

エージェントモード終了を、アラームで知らせる。

エージェントモードやワークフローモードの自動実行が完了した時に、アラームで通知できます。長い生成を待っている間も、完了タイミングを見逃しにくくします。

Sleep Prevention

AIエージェント生成中は、PCのシャットダウンやスリープを抑える。

AIエージェントが生成中の間はスリープ防止を有効にし、途中でPCが止まらないようにします。エージェントモード終了時には自動解除され、通常の状態へ戻ります。

Reasoning

リーズニング出力を、表示・非表示で切り替える

思考プロセスを確認したい時だけ展開し、通常は本文に集中できます。

Reasoning Hidden

リーズニング出力を折りたたみ、本文に集中する。

リーズニングモデルの思考欄は、必要ない時は非表示にできます。長い回答でも本文の読みやすさを保ちます。

Reasoning Visible

必要な時だけ、思考プロセスを展開する。

Thought for a moment を開くと、リーズニングモデルの内部出力を確認できます。回答の背景を見たい時だけ表示できます。

Prompt Position

プロンプトシナジーを起こすプロンプト配置

コンテキストファーストとプロンプトファーストで、今回の入力をどこに置くかをコントロールします。 選択中の配置は、生成後のアシスタント表示にも context first / prompt first として表示されます。

固定のシステム層は常に先頭に置き、その下の文脈領域で「今回の入力」の位置を切り替えます。 モデルに先に文脈を読ませるか、先に指示の方向を置くかを選ぶ、Local Nova の中核コントロールです。

Project Knowledge

独立して知識管理するプロジェクト機能

通常チャットとは別に、プロジェクト単位で知識ノード、ファイル、会話を管理できます。 必要な知識だけを読み込み、特定テーマの対話を独立した文脈として育てられます。

Project Manager

通常チャットとは別に、プロジェクトを管理する。

プロジェクトマネージャーでは、通常チャットとは独立した管理領域としてプロジェクトを作成できます。研究、制作、ライティングなど、テーマごとに知識と会話を分けて扱えます。

Knowledge Input

テキストやファイルを、プロジェクト知識として追加する。

知識入力では、使いたいテキストを貼り付けたり、txt、md、html、css、js、pdf、jpeg、png などのファイルを追加できます。追加した内容はプロジェクト内の知識ノードとして保存されます。

Knowledge Manager

読み込む知識を、必要なものだけ選ぶ。

知識管理では、追加済みノードを一覧で確認し、読み込み ON/OFF を切り替えられます。会話に使う資料だけを選ぶことで、文脈を絞ったプロジェクト対話ができます。

Project Chat

プロジェクト知識を使って、会話を継続する。

プロジェクト内のチャットでは、選択した知識ノードを補助文脈として使えます。資料の要約、調査、制作方針の整理などを、プロジェクトごとの独立した知識空間で進められます。

Pre Generate

思考能力を向上させるプレ出力

メイン出力の前に、思考段階を一度つくる。

リーズニングを持たない小規模モデルでも、論理思考と仮説探索をプレ出力で補強できます。

Pre Generate Mode

メイン出力の前に、思考段階の出力を挟む。

プレ出力は本回答の前に行う下準備の出力です。通常送信に加えて、CoT的な論理プレ出力と、ToT的な探索プレ出力を選べます。

Logical Pre Generate

既知のタスクは、CoT的に論理を積み上げる。

論理プレ出力は、タスク分解や手順整理に向いています。既知の課題では、回答前に必要な論点を整えてから本回答へつなげます。

Exploration Pre Generate

未知のタスクは、ToT的に仮説を広げる。

探索プレ出力は、答えが定まっていない課題に向いています。複数の視点や仮説を先に広げ、本回答の質を高めます。

Agent / Workflow

AIエージェント&ワークフロー

複数回の思考を、自動でつなげる。

エージェントモードでプロンプトを自動投入し、ワークフローモードで複数のプロンプトを順番に実行できます。

Agent Mode

エージェントループを組み、複数回プロンプトを自動入力する。

エージェントモードでは、AIの回答後に指定したプロンプトを自動で再投入します。改善、要約、視点追加などを複数回つなげ、短いモデルでも段階的に出力を磨けます。

Loop Count

自動実行の回数を、画面上で確認する。

エージェントモード中は、現在の実行回数と最大回数を表示します。どこまで自動ループが進んでいるかを、会話中にすぐ把握できます。

Green Active State

動作中は、画面周辺がグリーンに発光する。

Aボタンと画面外周の発光で、エージェントモードが機能している状態を強調します。通常会話との違いを視覚的に判断できます。

Small Model Loop

小規模モデルの軽さを、反復思考に活かす。

小規模言語モデルは大規模モデルよりノイズが少なく、文脈が濁りにくい場面があります。その強みを活かし、短いエージェントループで出力を段階的にブラッシュアップします。

Workflow Mode

プロンプトを順番に組み、作業手順として実行する。

ワークフローモードでは、複数のプロンプトをノードとして並べ、順番に自動実行できます。企画、スコアリング、要約、結論作成など、決まった手順を一つの流れにできます。

Workflow Templates

よく使うワークフローを保存し、あとから呼び出す。

作成したワークフローはテンプレートとして保存できます。繰り返し使う思考手順を呼び出し、必要に応じて上書きしながら、プロジェクトごとの型を育てられます。

Launch Flow

起動はシンプル。使い方は深く。

Node.js 標準機能だけで動く最小構成。Ollama を起動し、モデルを選んで、ブラウザから会話を始めます。

まずはZIPをダウンロード

GitHub Releases を開き、Assets の Source code (zip) をダウンロード。解凍後、フォルダ内のファイルから Local Nova を起動します。

Download ZIP

Ollamaを起動

ローカルで使うAIモデルの実行環境を先に立ち上げます。

モデルを用意

`ollama pull` で使いたいモデルを取得します。

Local Novaを起動

起動ファイル、またはプロジェクトに合わせた起動コマンドからアプリを立ち上げます。

ブラウザで開く

macOS

起動コマンドの例

# 1. アプリのフォルダへ移動（パスは自分の環境に合わせて変更）
cd /Users/名前/Downloads/localnova-1.0.0

# 2. アプリを起動
npm start

# 3. ブラウザで次のアドレスを開く
http://localhost:3001

「名前」は自分の Mac のユーザー名に置き換えてください。
ターミナルに「cd 」と入力したあと、解凍したフォルダをターミナルへドラッグ＆ドロップするとパスが自動入力されます。
事前に Ollama を起動しておく必要があります。アプリを終了するときはターミナルで Ctrl + C を押します。

macOS

起動ファイル（.command）を有効にする

# 1. 実行権限を付ける（最初の1回だけ）
chmod +x /Users/名前/Downloads/localnova-1.0.0/LOCAL_NOVA.command

# 2.「開発元が未確認」の警告を消す
xattr -d com.apple.quarantine /Users/名前/Downloads/localnova-1.0.0/LOCAL_NOVA.command

ダウンロードした起動ファイルは、そのままだと実行権限がなく macOS の警告も出ます。上記2行を最初の1回だけ実行すれば、以降は LOCAL_NOVA.command をダブルクリックするだけで起動できます（Ollama の起動・ブラウザ表示まで自動）。
「cd 」と同様に、コマンドのあとへファイルをドラッグ＆ドロップするとパスが自動入力されます。手順2で「No such xattr」と表示された場合は、すでに警告対象でないだけなので無視して問題ありません。

※ パス内の「localnova-1.0.0」のバージョン番号は、ダウンロードした版と完全に一致させてください（例：v1.1.0 をダウンロードした場合は「localnova-1.1.0」になります）。
※ パス内の「Downloads」は、zip をダウンロードフォルダに保存・解凍した場合の例です。別の場所に置いた場合は、その保存先のパスに置き換えてください（フォルダをターミナルへドラッグ＆ドロップすれば確実です）。

Concept & Context Engineering

軸を圧縮し、文脈を組み立てる。

Local Nova の設定群は、AIに渡す「軸」と「文脈」を設計するための操作面です。 短く凝縮したコンセプトで方向を定め、知識・記憶・配置で文脈環境を組み立てる ── コンセプトエンジニアリングとコンテキストエンジニアリングを、ひとつの画面で実践できます。

コンセプトエンジニアリング

AIに渡す「軸」を、短いフレーズへ圧縮して設計する考え方です。長い指示を積み重ねるのではなく、核となる一語・一句に思想を凝縮し、AIのアテンションを意図した方向へ収束させます。

Local Nova では、セッションプロンプト・カスタムプロンプト・システムプロンプトとして、効かせたい範囲ごとに「軸」を置けます。さらに派生モデルへ焼き付ければ、軸はモデルそのものになります。

コンテキストエンジニアリング

AIに渡す「文脈」の中身と並び順を設計する考え方です。どんな材料を、どの位置に置いて渡すかで、生成の質と一貫性が変わります。

Local Nova では、プロジェクト知識・他スレッドメモリー・ウェブ検索を文脈の材料として束ね、プロンプト配置（Context / Prompt First）で並び順まで設計できます。

この2つの設計を、具体的に支える設定群です。

思考と出力

  • コンテキスト2048〜32768
  • 出力形式一括 / 段階

モデル設計

プロンプトを「会話に渡す指示」で終わらせず、モデルそのものに焼き付けるところまで踏み込めます。

  • system prompt の二層編集 スレッド単位のセッションプロンプトと、全体共通のカスタムプロンプトを分けて編集し、指示の効く範囲を設計します。
  • ModelFile ビルダー 固定 SYSTEM を役割ごとに整理すると Ollama の ModelFile が組み上がり、ターミナル操作なしでモデル作成を準備できます。
  • 派生モデルの作成と切り替え SYSTEM を焼き付けた用途別のオリジナルモデルを保存。通常モデルと同じ一覧から、会話開始時にそのまま選べます。

表示と集中

UI を「思考に集中できる空間」として整えるための調整機能を備えています。

  • 10種類のテーマ 用途や気分に合わせて UI 全体の配色を切り替え、長時間でも疲れにくい見え方を選べます。
  • Focus モード サイドバーを畳んで対話領域を最大化。情報を絞り、いまの会話だけに集中できます。
  • 背景画像・透明度・エフェクト 背景画像や透明度、視覚エフェクトを調整し、思考の銀河を自分だけの作業空間として仕立てられます。
  • 出力中スクロール 生成中の自動追従の有無を選択。読みながら考えるか、流れを眺めるかを好みに合わせられます。
